# 인스타 릴스 챌린지 선별·랭킹 프롬프트

인스타그램 릴스의 **"오리지널 오디오" 기반 인기 후보 풀**을 LLM에 입력해, **챌린지/포맷이 뚜렷한 콘텐츠 10개를 직접 선별·랭킹하고 이름까지 작명**하는 프롬프트입니다. 제목이 모두 "오리지널 오디오"라 이름이 없는 상태에서, 영상 description의 공통점을 분석해 정체를 추론하는 것이 핵심입니다.

- **분야:** AI / 프롬프트
- **입력:** 합산 조회수 내림차순 후보 풀 JSON (후보당 description N개)
- **출력:** #1~#10 랭킹 (콘텐츠 이름 + 2줄 설명: 포맷 설명 / 원작자·선도 유저·어림 조회수)

## 두 가지 프롬프트

| 파일 | 역할 |
|---|---|
| `prompt_selection.md` | **선별형 (최신)** — 55개 후보 풀에서 챌린지성이 뚜렷한 10개를 LLM이 직접 선별 |
| `prompt_ranking.md` | **랭킹형 (이전)** — 확정된 10개를 작명·2줄 설명. 선별형이 이 작명 로직을 계승 |

## 선별 로직 (핵심)

1. **챌린지성 평가** — 후보별 `description` 동질성을 측정. 강→약 신호: 반복 해시태그 > 반복 키워드/문장 패턴 > 반복 포맷(비포애프터·리액션·도전 등) > 동일 소재
2. **범용 BGM 배제** — 조회수가 높아도 description 주제가 제각각인 "범용 배경음"은 제외 (조회수순으로 자르지 않음)
3. **순위 기준** — 챌린지성 1순위, 인기·확산(total_views·unique_user) 2순위
4. **작명** — 이름이 없으므로 description 공통점으로 한국어 작명(3~12자), 과장·날조 금지
5. **정직성** — 챌린지성 뚜렷한 후보가 10개 미만이면 억지로 채우지 않고 확보분만 출력

## 파일

| 경로 | 설명 |
|---|---|
| `prompt_selection.md` | 선별형 프롬프트 (System + User + 출력 예시) |
| `prompt_ranking.md` | 랭킹형 프롬프트 (이전 버전) |
| `data/` | 후보 풀 데이터 — 주차별 → 월 통합 → 후보55 → 설명10(LLM 입력) 집계 파이프라인 JSON |
