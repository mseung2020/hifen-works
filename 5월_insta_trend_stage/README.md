# 인스타 트렌드 단계 진단

인스타그램에 떠오르는 트렌드 키워드가 지금 **생애주기 어느 단계**(태동·점화·확산·정점·쇠퇴)에 있는지를 **일별 누적 포스트 그래프**로 보면서 자동 분류해주는 Flask 대시보드입니다. 최근 1주 증가량과 이전 1주 증가량을 비교해 단계를 판정합니다.

- **분야:** 데이터 분석
- **결과물 형태:** Flask 대시보드
- **대상:** 7개 카테고리(맛집/푸드, 뷰티, 엔터, 운동/피트니스, 육아/키즈, 카페/핫플, 패션/룩) × 카테고리별 트렌드 클러스터 30개

## 동작
1. 카테고리별로 클러스터 30개의 **일별/누적 포스트 수**를 조회 (내부 API)
2. 결과를 `cache/`에 날짜별로 캐싱 (재조회 비용 절감)
3. 최근 1주 증가량 vs 이전 1주 증가량을 비교해 단계 자동 판정
4. 누적 그래프 + 단계 라벨을 대시보드에 표시

## 실행

```
pip install flask requests
# 실시간 재조회가 필요할 때만 내부 API 설정 (캐시 열람만 하면 불필요)
export TREND_API_URL=... TREND_API_TOKEN=...
python app.py        # → http://localhost:5000
```

## 구성

| 경로 | 설명 |
|---|---|
| `app.py` | Flask 서버. 캐시 우선 조회, 없으면 내부 API에서 클러스터별 일별 통계 수집 |
| `templates/index.html` | 누적 그래프 + 단계 진단 대시보드 UI |
| `cache/` | **2026-05-20 스냅샷** — 7개 카테고리의 클러스터별 일별/누적 포스트 수 JSON (실제 분석 결과 데이터) |

> **보안:** 원본 `app.py`에 하드코딩되어 있던 내부 API URL과 JWT 토큰은 제거하고 환경변수(`TREND_API_URL`, `TREND_API_TOKEN`)로 분리했습니다.
>
> **참고:** 앱은 `cache/{카테고리}_{오늘날짜}.json`을 찾으므로, 포함된 2026-05-20 캐시로 그래프를 보려면 해당 날짜 기준으로 조회하거나 캐시 파일명을 당일 날짜로 맞추면 됩니다. 캐시 JSON 자체가 분석 결과 데이터입니다.
