# 클로드 소스코드 분석

Claude Code의 전체 소스 코드와 내부 지침(System Prompt)을 분석해 **Anthropic이 Claude를 어떻게 설계했는지** 구조를 파악하고, 이를 바탕으로 하이픈 자체 AI 챗봇 개발에 영감을 얻기 위한 연구 작업입니다. 비개발자도 이해할 수 있는 보고서 형태로 정리했습니다.

- **분야:** 백엔드 / 코드
- **분석 대상:** Claude Code v1.x 소스 코드 (`src/`)
- **도구:** 소스코드 구조 분석 + MCP 연결로 노션에 보고서 작성

## 결과물

[`Claude_Code_분석_보고서.md`](Claude_Code_분석_보고서.md) — 6개 챕터 보고서:

1. **Claude는 어떻게 '생각'하도록 만들어졌는가** — 규칙집이 아닌 '철학'으로 작동하는 설계, 실제 System Prompt 철학 발췌
2. **AI에게 역할과 도구를 주는 법** — System Prompt의 역할 부여, ALWAYS/NEVER 원칙, Tool Description 설계
3. **AI는 어떻게 '스스로 생각'하는가** — ReAct 루프, 멈추는 조건, 다중 에이전트 구조
4. **AI의 기억과 비용, 똑똑하게 관리하기** — Context Window, 토큰 과금 구조, Prompt Caching, 비용 최적화
5. **지금 당장 써먹을 수 있는 것들** — 5가지 핵심 원칙, 하이픈 챗봇 3단계 로드맵, System Prompt 템플릿
6. **번외: Claude Code만의 독특한 기술들** — 병렬 사전 실행, 배치 병렬, 스트리밍 선실행, 자동 압축, 에러 보류, 대용량 오프로딩, 2단계 AI 분류기, 투기적 사전 분류, 명령어 접두사 학습

> 분석 결과물은 원래 노션 문서로 작성되었으며, 이 레포에는 마크다운으로 변환해 포함했습니다.
>
> 분석 대상이었던 Claude Code 소스 코드는 Anthropic의 자산이므로 레포에 포함하지 않습니다.
