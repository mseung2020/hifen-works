# 릴리바이레드 브랜드 마케팅 대시보드 — 프로젝트 구조도

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 목적 | 브랜드 마케터를 위한 맞춤 마케팅 컨설팅 대시보드 (위기감 조성 → FOMO 해소) |
| 샘플 브랜드 | 릴리바이레드 (뷰티 & 메이크업) |
| 기술 스택 | Python Flask + Pandas + HTML/CSS/JS + Chart.js |
| 서빙 | 로컬 Flask 서버 (localhost:5000) + Cloudflare Tunnel로 외부 공유 |

## 3. 데이터 가공 흐름 (app.py)

```
CSV 10개 로드
    │
    ├─► [KPI 산출]
    │     릴리바이레드.csv → 총 영상 수, 총 조회수, 광고/오가닉 비율
    │     youtuber_stats.csv → 평균 참여율
    │
    ├─► [경쟁사 분석]
    │     brand_all_videos.csv
    │       → brand1 컬럼 value_counts() → 브랜드별 협업 영상 수 랭킹
    │       → 릴리바이레드 순위 산출 (19위 / 1,161개 브랜드)
    │       → 크리에이터 겹침 계산: lily_channels ∩ brand_channels
    │       → 히트맵 매트릭스: 8×8 브랜드 간 겹침 수
    │
    ├─► [크리에이터 분석]
    │     youtuber_info + youtuber_stats + 릴리바이레드.csv + brand_all_videos
    │       → JOIN on channel_id
    │       → 크리에이터별: 구독자, 평균조회, 참여율, 경쟁사 협업 목록
    │       → 구독자 내림차순 상위 20명 추출
    │
    ├─► [시청자 인구통계]
    │     demography.csv
    │       → F/M 14개 연령 구간 평균 → 여성/남성 비율, 연령 분포
    │
    ├─► [올리브영]
    │     oliveyoung_brand_score.csv
    │       → brand ID → 브랜드명 매핑 (BRAND_ID_MAP)
    │       → 5개 브랜드 일별 스코어 시계열 데이터
    │     oliveyoung_product_rank.csv + oliveyoung_product_info.csv
    │       → 릴리바이레드 제품만 필터 → 최신 날짜 순위 + 전일 대비 변동
    │
    └─► JSON 직렬화 → index.html 템플릿에 전달
```

## 4. 프론트엔드 구조 (index.html)

```
index.html (단일 파일)
    │
    ├─► [랜딩 페이지] — 3가지 버전 선택 카드

    │
    ├─► [Version A: Clean Minimal]
    │     상단 네비 → 히어로(KPI 5개) → 경쟁사 바차트 + 랭킹
    │     → 올리브영 라인차트 + 제품순위 → 크리에이터 카드 8개
    │     → 오디언스(도넛+연령바+광고비율)
    │
    ├─► [Version B: Data-Dense Dashboard]
    │     사이드바 → 스코어카드 4개 → 히트맵 + 랭킹
    │     → 크리에이터 리스트(경고태그) + 오디언스
    │     → 올리브영 라인차트 + 제품순위
    │
    ├─► [Version C: Storytelling]
    │     풀페이지 스크롤 (5개 섹션 + CTA)
    │     왼쪽 dot 네비 = IntersectionObserver로 자동 활성화
    │     Hero → VS비교 + 수평바 → 크리에이터 카드 3개
    │     → 인구 피라미드 + 인사이트 → 올리브영 → CTA
    │
    └─► [공통]
          Chart.js (CDN) — 바, 라인, 도넛 차트
          플로팅 홈 버튼 — position:fixed, 우측 하단
          chart-wrap div — 차트 높이 고정 (무한 늘어남 방지)
```

## 5. AI(Claude)가 수행한 작업 상세

### 5-1. 데이터 분석 & 기획

| 단계 | AI가 한 일 |
|---|---|
| 스키마 파악 | scheme.csv(한 줄짜리)를 파싱하여 scheme_table.csv로 변환 (330개 테이블, 4,915개 컬럼 정리) |
| 데이터 탐색 | 릴리바이레드.csv 구조 파악 (14개 컬럼, 646행, 주요 크리에이터/영상 특징 분석) |
| 대시보드 기획 | 7개 섹션 기획안 작성 (Brand Health, Competitor Radar, Creator Intelligence 등) + 각 섹션별 필요 테이블 매핑 |
| 경쟁사 정의 전략 | "뷰티 전체"가 아닌 "같은 크리에이터를 공유하는 브랜드"로 경쟁사 자동 탐지 전략 제안 |

### 5-2. SQL 작성 & 데이터 수집 지원

| 단계 | AI가 한 일 |
|---|---|
| SQL 작성 | 10개 쿼리 작성 (queries_실행용.sql) — 367개 channel_id IN절 자동 생성 포함 |
| 에러 대응 | 요청1 실행 시 `Unknown column 'b.subscribers'` 에러 → 스키마 재확인 후 수정 |
| ID 매핑 발견 | 올리브영 brand 컬럼이 이름이 아닌 숫자 ID임을 파악 → brand_logo_beauty.csv에서 ID 매핑 추출 → 쿼리 수정 |
| 추가 데이터 제안 | 기본 4개 → youtuber_info, data_mining 추가 제안 → 올리브영 3개 테이블 추가 제안 |
| 데이터 검증 | 각 CSV 다운로드 후 행 수, 컬럼 구조, 핵심 통계 확인 |

### 5-3. 디자인 & 프론트엔드

| 단계 | AI가 한 일 |
|---|---|
| 목업 제작 | 3가지 버전 HTML 목업 작성 (A: Minimal, B: Dashboard, C: Storytelling) — 플레이스홀더 데이터로 디자인 검증 |
| 통합 구현 | 3개 버전을 단일 index.html에 통합, 랜딩 페이지 + 탭 전환 구조 |
| Chart.js 연동 | 7종 차트 구현 (바, 라인, 도넛, 가로바, 스택바, 히트맵 테이블, CSS 피라미드) |
| 차트 높이 버그 수정 | Chart.js canvas 무한 확장 문제 → chart-wrap div로 고정 높이 적용 |
| 홈 버튼 수정 | 아이콘 정중앙 배치 (align-items, justify-content, line-height) |
| 로고 변경 | "DYD analytics" → "유광기"로 전체 교체 |
| 레이아웃 수정 | Version A 히어로 가로폭을 콘텐츠와 통일 (max-width:1400px, 가운데 정렬) |
| 오디언스 정렬 | Version B 오디언스 차트 세로 가운데 정렬 |
| 올리브영 간격 | Version C 제품 순위 #번호와 텍스트 겹침 해소 |
| dot 네비게이션 | Version C 스크롤 시 왼쪽 dot 자동 활성화 (IntersectionObserver) + 클릭 시 스크롤 이동 |

### 5-4. 백엔드 (app.py)

| 단계 | AI가 한 일 |
|---|---|
| Flask 서버 | app.py 작성 — CSV 10개 로드, 데이터 가공, JSON 직렬화, 템플릿 서빙 |
| 데이터 가공 | 8개 분석 모듈 (KPI, 경쟁사, 크리에이터, 인구통계, 올리브영, 월별트렌드, 상위영상) |

| 한국식 포맷 | format_subs_kr() — 구독자 수를 "45.9만" 형태로 변환 |
