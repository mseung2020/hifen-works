# 인스타 트렌드 분석 워크플로우 프롬프트

인스타그램·유튜브 콘텐츠 데이터에서 **카테고리별 TOP K 트렌드 클러스터**를 추출하고, 각 클러스터에 대해 **광고주 인사이트**와 **트렌드 스토리**를 자동 생성하는 Dify 워크플로우입니다. 마케팅 의사결정용 LLM 산출물을 신뢰성 있게 뽑아내는 것이 목표입니다.

- **분야:** AI / 프롬프트
- **플랫폼:** Dify (AI 워크플로우)
- **모델 구성:** gpt-5-mini(1차 생성) + Claude Sonnet 4.5(자기검증) + Tavily(웹서치)
- **입력:** 인스타/유튜브 CSV + 카테고리 힌트 + TOP K
- **출력:** `advertiser_insight_final`, `trend_story_final` 등 8개 변수 JSON

## 두 가지 산출물

| 산출물 | 내용 | 하네스 |
|---|---|---|
| **광고주 인사이트** | 왜 뜨나 / 누구에게 통하나 / 캠페인에 어떻게 쓸까 | 단일생성 + 자기검증 1패스 |
| **트렌드 스토리** | 점화 → 확산 경로 → 현재 단계(7단계 enum) 서사 | 단일생성 + **결정론적 Python 사실검증** + 자기검증 |

## 신뢰성 메커니즘 (설계 핵심)

1. **결정론적 사실 검증** — 트렌드 스토리가 인용한 `cited_facts`를 Python이 입력 데이터(`cluster_pack`)와 직접 매칭(핸들 부분일치, 날짜 정확일치, 수치 ±2%). LLM 자기검증이 놓치는 환각을 구조적으로 차단
2. **`availability_flags` 패턴** — 모든 옵셔널 데이터에 boolean 플래그를 두고, 프롬프트가 "flag=false면 추측 금지"로 분기 → 결측 데이터 환각 차단
3. **카테고리 무관 설계** — 프롬프트에 "맛집/패션" 같은 하드코딩 없음. `category_hint` 변수로 LLM이 자기 문맥화, 자기검증이 하드코딩 검출
4. **정직한 결측 처리** — 데이터 한계는 `data_caveats` 배열에 기록

## 파일

| 경로 | 설명 |
|---|---|
| `prompts/` | 5단계 프롬프트 (클러스터 추출 보조 서브키워드 / 광고주 인사이트 1차·검증 / 트렌드 스토리 1차·검증) |
| `dify_workflow.yml` | Dify 워크플로우 export (v8) |
| `인계노트.md` | 아키텍처·데이터 계약(cluster_pack 스키마)·LLM 스펙·하네스 설계 의도·인계 문서 |
| `data/` | 워크플로우 출력 — 카테고리별 클러스터 분석 결과 JSON (맛집_푸드 45 / 뷰티_헤어 6 / 엔터 8) |
| `reports/` | 결과물 HTML 리포트 (종합 + 카테고리별, 자체 완결형) |
| `examples/` | 광고주 인사이트·트렌드 스토리 출력 예시 이미지 |
