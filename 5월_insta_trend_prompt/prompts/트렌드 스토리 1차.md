[모범 예시 — 구조·밀도·어체를 따르세요. 내용(계정명·수치·카테고리)은 입력 데이터에서 가져오세요]



ignition narrative:
"[신제품/이벤트명]의 [플랫폼] 출시가 [핵심 포맷·화법]과 결합하며 카테고리 관심 집중."
또는 "[@계정]의 [콘텐츠 유형]이 [핵심 요소·화법]을 각인시키며 카테고리 점화."
또는 "[관찰된 패턴]이 [플랫폼]에서 [확산 방식]으로 반복되며 카테고리 관심 집중."
→ 1문장. 명사형 종결. 점화 계기 + 점화 메커니즘.



spread narrative:
"[최초 계정/채널]의 [포맷A]에서 [후속 계정/채널]의 [포맷B]로 형식 전환, [정성적 임팩트 표현] 기록됨. [플랫폼]에서는 [파생 콘텐츠 유형]으로 분기하며 확산 경로 다변화됨."
→ 2문장. 마침표로만 구분. '누가 → 어떤 형식으로' 경로 변형 명시.



current_stage narrative:
"[게시물 추이 방향 + 핵심 신호 해석]으로 [단계] 진입 확인됨."
→ 1~2문장. 수치의 의미를 해석할 것 — 단순 수치 나열 금지.
→ "카테고리 평균" 비교는 데이터에 해당 값이 있을 때만 쓸 것. 없으면 수치 자체의 의미를 해석.
→ 파생 SKU·모방 진입·밈화 등의 신호가 있더라도, 게시물 추이가 하락 중이라면 쇠퇴 단계로 판단할 것.

쇠퇴 narrative 예시:
"제철 종료와 맞물려 피크 대비 게시물 생산이 급감하고 신규 계정 진입이 거의 없어 쇠퇴 초기 진입 확인됨."
"피크 이후 2개월 이상 게시물 감소가 지속되고 동일 포맷이 반복 소비 단계에 진입하며 쇠퇴 중기 확인됨."



advertiser_recommendation:
→ 현재 단계에 맞는 행동 권고. 1~2문장. 명사형 종결.
- 태동·점화: 선점 진입 권고 위주
- 확산: 차별화된 조기 진입 권고
- 정점 초기: 진입 적기 + 차별화 조건 제시
- 정점 중기·후기: 포화 시작, 명확한 차별점 없는 진입 비권고 + 차별화 전략 제시
- 쇠퇴 초기: 차별화 없는 진입 비권고 + 파생 서브포맷·피벗 방향 제시
- 쇠퇴 중기·후기: 카테고리 이탈 또는 파생 서브포맷 공략 권고

[섹션 1: 점화]
narrative: 정확히 1문장. 명사형 종결.



✅ 점화 주체 판단 — 아래 우선순위 순으로 판단할 것:



① 신제품·콘텐츠·이벤트 출시가 트렌드 계기인 경우 (최우선):
   content.instagram description에서 아래 유형을 언급하는 게시물이 단기간 집중되면
   → 해당 상품/콘텐츠/이벤트를 점화 계기로 삼을 것.
   · F&B: 편의점 출시·신메뉴·팝업 스토어·한정 출시
   · 뷰티·헬스: 신제품 론칭·올리브영 입점·한정판 출시
   · 패션: 브랜드 컬렉션 공개·아이돌 착용·콜라보 출시
   · 엔터: OTT 공개·음원 발매·시즌 오픈·아이돌 활동 재개
   · 여행: 항공 노선 취항·비자 정책 변경·드라마 배경지 화제
   · 육아: 신제품 공동구매 오픈·맘카페 바이럴·브랜드 인증 획득
   - date_or_period: 해당 클러스터의 최초 게시일
   - primary_actor: 상품명·콘텐츠명·이벤트명 (예: "GS25 피프레도 출시", "장원영 라이브 언급", "넷플릭스 OO 공개")
   - narrative: "[상품/이벤트]의 [플랫폼] 출시가 [핵심 포맷·화법]과 결합하며 카테고리 관심 집중."



② 단일 고임팩트 게시물이 있는 경우:
   다른 게시물 대비 조회수·likes·followers가 압도적으로 높은 단일 포스트가 있으면
   → 해당 계정과 날짜를 사용.
   - narrative: "[@계정]의 [콘텐츠 유형]이 [핵심 요소·화법]을 각인시키며 카테고리 점화."
   - 계정명은 언급 가능하나, 반드시 @ 기호를 앞에 붙일 것. 팔로워 수·DM 수·조회수 등 원시 수치는 함께 쓰지 말 것.



③ 일상적 카테고리·점진적 트렌드인 경우 (①②가 모두 해당 없을 때):
   특정 계정을 억지로 점화 주체로 지정하지 말 것.
   - primary_actor: "다수 계정" 또는 주요 해시태그·콘텐츠 군으로 표기
   - date_or_period: 데이터 내 최초 날짜가 아닌, 최근 게시물이 집중된 기간으로 표기
   - narrative: "[관찰된 패턴]이 [플랫폼]에서 [확산 방식]으로 반복되며 카테고리 관심 집중."



date_or_period 표기 규칙 (섹션별 형식이 다름):
- 형식은 반드시 `YYYY.MM.DD` (점 구분, 2자리 월/일). "YYYY년 M월" 월 단위 표기 금지.
- ignition: 단일 날짜 `YYYY.MM.DD` (점화 게시물·신제품·이벤트가 발생한 하루)
- spread: 기간 `YYYY.MM.DD~YYYY.MM.DD` (점화 다음날부터 확산 정점일까지)
- current_stage: `YYYY.MM.DD~진행중` (확산 정점 다음날부터 데이터 마지막 시점까지, 종료는 "진행중"으로 표기)
- 세 기간은 서로 겹치지 않고 하루씩 연속될 것 (ignition일 → 다음날 spread 시작 → spread 종료 다음날 current_stage 시작).
- 모든 날짜는 content.instagram[].publish_date 와 content.youtube[].publishDate 분포를 근거로 판단할 것.



primary_actor 표기 규칙:
- 핵심 주체명 또는 이벤트명만 간결하게. "및"·"with" 등으로 복수 주체를 연결하거나 부연 설명을 붙이지 말 것.
  예) "LE SSERAFIM 'CELEBRATION' MV" (O) / "LE SSERAFIM 'CELEBRATION' 공식 MV 및 정규 2집 선공개 발매" (X)
- ③ 케이스: "다수 계정" 단독 또는 "다수 [콘텐츠 유형] 계정" 수준으로 표기. 특정 브랜드·계정명 추가 금지.



❌ 쓰지 말 것:
- 조회수 0, dm_send_count null 등 임팩트 없는 최초 게시물 계정을 점화 주체로 기계적 선택.
- 웹컨텍스트(뉴스 기사·블로그·해외 매체)의 날짜나 출처를 점화 시점/주체로 쓰는 것.
  trend_meta는 배경 맥락 참고용이며 점화 날짜 판단 소스가 아님.
- primary_actor가 뉴스 매체·블로그·웹사이트(예: 네이버 블로그, NYT)이면
  → content.instagram 또는 content.youtube의 SNS 계정명·신제품명·이벤트명으로 교체.
- narrative 안의 플랫폼·채널 유형이 primary_actor와 불일치하면 수정.
  예) primary_actor가 인스타그램 계정인데 narrative에 "유튜브 채널" 표현 → "인스타그램 게시물"로.
- 두 번째 문장 절대 금지. 후속 확산 수치는 spread에서 다룰 것.



[섹션 2: 확산 경로]
narrative: 정확히 2문장. 명사형 종결.



✅ 써야 할 것:
- '누가 → 어떤 형식으로' 변형됐는지 경로 명시
- 두 문장이 각각 다른 채널의 수치만 나열하는 방식 금지
  두 문장 중 적어도 하나는 포맷 변형(리뷰→DIY, 이미지→영상 등)을 서술할 것
- 임팩트는 원시 수치 대신 정성적 표현으로 ("폭발적 도달" "단기 집중 확산" 등)
- content.youtube 영상이 있으면: 채널명과 포맷 변화 중심으로 서술 (구독자·조회수 수치 직접 노출 금지)
  → youtube 데이터가 없으면 content.instagram 게시물 데이터만으로 서술.



❌ 쓰지 말 것:
- "→" 기호, ";" 세미콜론, "하고," "하며," "~이며," "~가운데," "~상황에서," 접속형으로 절을 연결하는 방식
  반드시 마침표(".")로만 문장을 구분할 것.
  이 접속형으로 연결된 절도 각각 별도 문장으로 카운트 — 합산 3절 이상이면 핵심 2개만 남길 것.
- 경로 분기가 여럿이면 임팩트 가장 큰 경로 하나만 서술
- 팔로워 수, DM 건수, 조회수 등 원시 수치를 숫자로 직접 노출하는 것



[섹션 3: 현재 단계]
phase_label: 아래 15개 중 하나. [단계 시기] 형식으로 출력.
- 태동 초기 / 태동 중기 / 태동 후기
- 점화 초기 / 점화 중기 / 점화 후기
- 확산 초기 / 확산 중기 / 확산 후기
- 정점 초기 / 정점 중기 / 정점 후기
- 쇠퇴 초기 / 쇠퇴 중기 / 쇠퇴 후기



단계 판단 기준:
- 태동: 소수 계정에서만 관찰, 협찬 없음, 확산 신호 미약
- 점화: 특정 계기(신제품·셀럽·이벤트)로 게시물 급증 시작
- 확산: 다수 계정에 퍼지고 포맷·채널 다변화 진행 중
- 정점: 최대 도달, 파생 유통·콘텐츠·브랜드 진입 활발
- 쇠퇴: 신규 진입 둔화, 양산화·포화 신호



시기 판단 기준:
- 초기: 해당 단계 진입 직후 (신호 강화 중)
- 중기: 해당 단계 특성이 안정적으로 유지됨
- 후기: 다음 단계로의 전환 신호 시작



narrative: 1~2문장. 수치를 나열하는 게 아니라 그 수치가 왜 이 단계인지 해석할 것.
   ❌ "총 조회 3,700만 회와 DM 66만 건이 발생함." (볼륨 나열만, 원시 수치 직접 노출)
   ✅ "전환 의향이 높은 협찬 콘텐츠가 단기간 집중되고 프랜차이즈 채널 진입이 겹치며 확산 중기 진입 확인됨." (해석)
trend_meta.advertiser_entry_timing의 판단을 참고하되, content 게시물 날짜 분포·신제품 등장 시그널로 검증할 것.



advertiser_recommendation: 1~2문장. 지금 이 단계에서 해야 할 행동. 명사형 종결.

[날짜 활용 규칙]
날짜는 각 섹션의 date_or_period 필드에만 표기할 것.
narrative 본문 안에는 어떠한 형태의 날짜(YYYY.MM.DD, "YYYY년 M월", "M월 N일", "지난주" 등)도 쓰지 말 것.
- ❌ "2026.05.05부터 다수의 체인·편의점 SKU 출시와..." → 날짜 제거
- ❌ "2026.05.04 정점 형성 이후 공식 MV..." → 날짜 제거
- ✅ 시점 정보가 필요하면 정성적 표현으로 대체: "정점 직후", "프로모션 종료 직후", "점화 이후 단기간 내", "확산 정점 이후" 등.
- 날짜 판단의 근거는 여전히 content.instagram[].publish_date 와 content.youtube[].publishDate 분포이지만, 그 결과는 date_or_period에만 반영하고 narrative에는 표면화하지 말 것.



[기간 판단 가이드]
- ignition일: 점화 계기로 판단된 단일 게시물·이벤트의 publish_date (시각 부분은 버리고 날짜만).
- spread 정점일: 점화 이후 게시물 일별 빈도(또는 임팩트 합산)가 가장 높았던 날. 이 날까지를 spread 기간으로.
- current_stage 시작일: spread 정점 다음날. 종료는 항상 "진행중".
- 정점 판단 근거가 모호하면 게시물 밀도가 처음 둔화되기 시작한 직전 날을 정점으로 본다.



[섹션 4: 요약]
narrative: 1문장. "~에요" "~이에요" "~있어요" 계열 어체.
현재 트렌드의 핵심 상태를 종합적으로 요약: 어떤 신호들이 맞물리고 있는지 + 어느 방향으로 전개되는지.
current_stage 내용을 더 읽기 쉬운 서사 형태로 압축하되, phase_label을 자연스럽게 녹여낼 것.



예시:
"올리브영 단독 기획, 올영픽 선정, 포켓몬 콜라보가 동시에 맞물리며 정점 단계에 진입했어요."
→ 첫 문장: 현재 단계를 만들고 있는 핵심 신호 나열 + 단계 진입 언급



❌ 쓰지 말 것:
- "~했음." "~기록됨." 명사형 종결
- current_stage narrative를 그대로 반복하는 것
- 원시 수치 직접 노출

[전체 금지 사항]
1. 모든 문장은 "~했음." "~기록됨." "~점화." "~필요." 명사형·과거형으로 종결.
   "~에요" "~입니다" "~이다" "~하라" "~합니다" "~하세요" 어체 금지.
2. 본문에 유보 표현 금지: "~일 수 있어요" "~로 추정" "확인이 필요" "가능성이 있음"
   → data_caveats에만.
3. 입력 데이터에 없는 수치·날짜·계정명 금지.
4. 각 narrative는 마침표 기준 2문장 초과 금지.
5. 팔로워 수·DM 건수·조회수·좋아요 수 등 원시 수치를 숫자로 직접 노출 금지.
   정성적 표현("단기 폭발적 도달" "전환 신호 농도 높음" "소형 계정 주도" 등)으로 대체.
