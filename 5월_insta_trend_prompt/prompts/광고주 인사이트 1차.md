why_trending 예시:
"[점화 계기: 특정 인물/이벤트]가 [플랫폼]에서 [콘텐츠]를 올리며 '[핵심 조합/요소]'가 소비자에게 각인됐어요. 이 [핵심 요소]가 기존 [카테고리]와 결합한 변형 포맷으로 [기간]부터 확산되며 지금의 형태로 자리잡고 있어요."
→ 2문장. "~에요" 어체. 배경 설명 아닌 메커니즘.
→ 두 문장은 독립된 근거 나열이 아니라 한 호흡으로 이어지는 흐름(점화 → 지금의 형태가 된 이유)이어야 함.
→ 두 문장의 종결어미를 동일하게 반복하지 말 것 (예: "~때문이에요"를 두 번 쓰지 말 것). "~됐어요/~고 있어요/~이에요/~자리잡고 있어요" 등을 자연스럽게 변주.


who_resonates 예시:
"[연령대] [계층명]이에요. [이 트렌드에서 관찰된 구체적 행동 패턴을 한 호흡으로 서술]. [효율 지표 또는 매체 접점 시그널을 자연스럽게 마무리]."
→ 마침표로 끊는 3단문. 첫 단문 반드시 연령대+계층으로 시작.
→ 세 문장의 종결어미를 모두 동일하게 쓰지 말 것 ("~이에요"를 3번 반복 금지). "~이에요/~해요/~보여요는 금지지만 ~이 뚜렷해요/~로 이어지고 있어요/~가 핵심이에요" 등으로 변주.
→ 세 문장의 길이(호흡)는 비슷하게 맞춰 따로 노는 인상을 피할 것.
→ why_trending 한 문장은 who_resonates 한 문장보다 길게 — 두 섹션 전체 분량은 비슷해 보이도록 균형.

[섹션별 작성 규칙]


[섹션 1: 왜 뜨고 있나]
정확히 2문장. "~에요" "~이에요"로 끝낼 것.
두 문장의 종결어미를 동일하게 반복하지 말 것 — 특히 "~때문이에요"를 연속 두 번 사용 금지.
두 문장은 근거 나열이 아니라 한 흐름(점화 → 지금의 형태가 된 이유)으로 자연스럽게 이어질 것.
한 문장의 길이는 who_resonates의 한 문장보다 길게 잡아 두 섹션의 전체 분량이 비슷해 보이도록 할 것.


문장 호흡 규칙(중요):
- 한 문장 안에 종속절·접속절은 최대 2개까지. "~했고, ~며, ~한 것이 ~이에요" 처럼 절이 3개 이상 중첩되는 구조 금지.
- "A하고, B하며, C해서, D인 것이 E이에요" 같이 연결어미가 3회 이상 이어지는 문장 금지.
- 한 문장이 화면에서 3줄 이상으로 늘어진다면 절을 줄이거나 두 문장으로 정보를 재배분할 것.
- 점화→확산→현재형태를 한 문장에 모두 욱여넣지 말고, 핵심 한 축만 첫 문장에서 다루고 나머지는 두 번째 문장으로 분리.


✅ 써야 할 것: '이 시점에 이 형태로 한국에서' 뜬 구조적 메커니즘
   점화 계기 → 확산 구조 → 지금 이 형태가 된 이유의 연결 고리
   trend_meta.why_trending을 핵심 근거로, content.instagram 게시물의 해시태그·화법 패턴으로 보완할 것


❌ 쓰지 말 것: "바이럴이 증폭됐다" "많은 사람이 관심을 가졌다" 같은 현상 묘사
   "~적으로 확산됐어요" "~노출이 이어진 상태예요" 같은 결과 서술만 있는 문장
   동일 종결어미를 두 문장에 반복하는 것 (특히 "~때문이에요" 연속)


[섹션 2: 누구에게 통하나]
마침표로 끊는 3단문. 각 단문은 한 줄 이내.
세 문장의 종결어미를 모두 동일하게 쓰지 말 것 — "~이에요"만 세 번 반복하는 형태 금지.
허용 변주 예: "~이에요", "~로 이어지고 있어요", "~가 뚜렷해요", "~가 핵심이에요", "~로 나타나고 있어요".
세 문장의 길이(호흡)를 비슷하게 맞춰 한 문단으로 자연스럽게 읽히게 할 것.


반드시 이 순서:
① 첫 단문: 연령대 + 계층 정의 — trend_meta.audience에서 추론할 것. 없으면 행동 기반 계층으로 대체.
   단순히 "20·30대 여성층이에요." 처럼 명사만 던지고 끝내지 말 것 — 그 계층을 규정짓는 핵심 속성(소비 동기·관심사·라이프스타일·정체성 등) 한 가지를 함께 포함해 2·3번째 문장과 호흡(길이)이 비슷해지도록 할 것.
   예) "20·30대 여성 다이어터층이에요." (X — 너무 가벼움)
       "건강·체중 관리에 민감하고 제철 식재료를 적극적으로 시도하는 20·30대 여성층이에요." (O)
② 둘째 단문: content.instagram 게시물에서 관찰된 구체적 행동 패턴 (협찬 화법, 해시태그 조합, 게시물 유형 등)
③ 셋째 단문: 효율 지표 또는 매체 접점 시그널. "~냄." "~보임." 명사형 가능.
   → content.instagram 게시물의 followers 값 분포로 팔로워 규모 유형(마이크로·중형·대형)을 파악하되,
     실제 수치(팔로워 수·DM 건수·조회수)는 본문에 직접 쓰지 말 것.
   → 이 트렌드에서만 관찰되는 고유한 효율 신호를 서술할 것.
     팔로워 규모 분포, 협찬 밀도·비중, dm_send_count 패턴, 저장·CTA 방식, 해시태그 조합 특성 중
     이 데이터에서 실제로 두드러지는 한 가지를 골라 구체적으로 표현.
   → "마이크로 계정 중심으로 협찬 확산 주도, 중형 계정이 최대 도달 담당" 형태의 문장을
     데이터 확인 없이 공식처럼 반복 사용 금지.
     그 패턴이 실제로 관찰된 경우라면, 왜 그런지(협찬 비중이 높다·저장 CTA가 집중된다 등)를
     함께 서술할 것.
   → dm_send_count가 없거나 null이 대부분이면: 협찬 게시물 비중·해시태그 패턴만으로 서술.


❌ 쓰지 말 것: 시기·기간을 첫 단문에 넣는 것 / 연령대 없이 시작하는 것
   "~편이에요" "~양상이에요" "~경향이에요" "~관찰돼요" "~나타나요" "~보여요" 어체


[섹션 3: 캠페인에 어떻게 쓸까]
items 정확히 3개. 각 item: header(항목명) + body(한 문장 행동 지시).


header 설정 원칙:
- 이 트렌드·데이터에서 광고주가 지금 당장 결정해야 할 가장 중요한 의사결정 축 3개를 직접 설정할 것.
- 예시에 나온 헤더("타겟팅 조정" "크리에이티브 테스트" "브랜드 세이프티")를 그대로 쓰는 것은 금지.
- 아래 참고 풀에서 이 트렌드·카테고리에 가장 적합한 방향을 고르되, 풀에 없는 새로운 헤더도 자유롭게 설정 가능:
  · 진입 타이밍: 채널 선점 타이밍 / 시즌 연동 / 프로모션 기간 활용 / 파생 SKU·라인업 선점 / 피크 전 선제 적재
  · 콘텐츠·크리에이티브: 포맷 A/B 전략 / 시리즈 콘텐츠 설계 / UGC 루프 설계 / 셀럽 연계 서사 / 성분·기능 비교 포맷
  · 유통·채널 (카테고리에 맞게 선택):
    F&B → 편의점 연동 / 오프라인 팝업 / 자사몰 연계
    뷰티·헬스 → 올리브영 입점 / 드럭스토어 채널 / D2C 전환
    패션 → 무신사·지그재그 기획전 / 팝업 스토어 / 시즌 컬렉션 연동
    엔터·여행 → OTT·플랫폼 연계 / 성지 거점 협업 / 패키지 연동
    육아 → 맘카페 공구 연계 / 안전인증 서사 / 육아 커뮤니티 진입
  · 브랜드 포지셔닝: 원산지·스토리 서사 / 차별 성분·기능 각인 / 경쟁 포지셔닝 / 커뮤니티 선점 / IP 연계
  · 리스크·전환: 밈화 방어 / 트렌드 주기 대응 / 팬덤 반감 방지 / 구매 전환 경로 단일화
- 3개의 헤더는 서로 다른 의사결정 차원을 다룰 것 (예: 타이밍+콘텐츠+유통 조합, 진입+포맷+포지셔닝 조합 등)


body 작성 규칙:
- 마침표 기준 정확히 1문장. 2문장 이상 금지.
- 이 트렌드 데이터에서 관찰된 구체적 신호(협찬 패턴, 해시태그, 유통 채널, 콘텐츠 포맷 등)를 근거로 작성
- 원시 수치 직접 노출 금지 — 정성적 표현으로 대체
- "~하기" "~사용." "~운영." "~설계." 명사형으로 끝낼 것
- "~하세요" "~주세요" "~하라" 어체 금지
- 막연한 지시 단독 사용 금지 (반드시 이 트렌드에 맞는 구체 조건이 따라와야 함)
- trend_meta.advertiser_entry_timing의 타이밍 판단을 반영할 것

[전체 금지 사항]
1. 본문에 유보·한계 표현 금지: "~일 수 있어요" "~로 추정" "확인이 필요" "~에 기반하므로"
   → 해당 내용은 data_caveats에만.
2. 입력 데이터에 없는 수치·날짜·계정명 금지.
3. "~습니다" "~합니다" "~됩니다" 어체 금지.
   why_trending · who_resonates 본문에 "~함." "~됨." 명사형 종결도 금지 — "~에요" 계열 어체로 써야 함.
   단, 한 섹션 안의 모든 문장을 같은 종결어미로 끝내는 것도 금지 — 어미를 변주해 한 호흡으로 읽히게 할 것.
4. 원시 수치(팔로워 수·DM 건수·조회수·좋아요 수 등) 본문 직접 노출 금지.
   정성적 표현(마이크로 계정 주도, 높은 전환 신호, 중형 계정 도달 등)으로 대체.
5. 계정명(user_id)은 해당 계정이 트렌드의 핵심 점화·확산 주체임이 명확한 경우에만 언급.
   단순 수치 예시·보조 근거 목적으로 괄호 안에 계정명을 언급하는 것 금지 — 행동 패턴 설명으로 대체.
   언급 시 반드시 @ 기호를 앞에 붙일 것 (예: @account_id).
6. 문장 호흡: 한 문장 안에 절과 수식어가 과도하게 중첩되지 않도록. 핵심 정보만 남기고 간결하게.
