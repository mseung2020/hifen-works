# 오늘자 광고 시장 1줄 어그로 헤드라인 5개 — 기획/설계서

> 작성일 2026-06-02 · 범위: **설계만** (코드는 다음 단계) · 최종 산출물: **1줄 한국어 헤드라인 5개(텍스트)**

---

## 1. 목표 한 줄

YouTube 인스트림(범퍼) 광고 원본 데이터를 조인·집계해 **"수집 데이터 기준 가장 최신 1일에 무슨 일이 있었나"**를 뽑고,
그 사건들에 LLM이 어그로 옷을 입혀 **마케터를 후킹하는 1줄 헤드라인 5개**를 만든다.

핵심 원칙: **집계·랭킹·이상치 탐지는 코드가, 작문(어그로)은 LLM이.**
LLM에 raw 행을 던지지 않는다. "오늘의 사건 후보 리스트(수치+맥락)"만 던진다.

---

## 2. 데이터 지형도

조인 중심축: **`video_id`** (영상 단위), **`channel_id`** (채널/브랜드 단위).

| 그룹 | 테이블 | 키 | 역할 |
|---|---|---|---|
| 영상 메타 | `YT_bumper_ad`, `YT_bumper_ad_global` | video_id, channel_id | 제목·브랜드(brand1)·장르·duration·ad_type·on_air·country |
| AI 콘텐츠 분석 | `YT_bumper_ad_analized;`, `YT_bumper_scene` | video_id | speech·ad_story·**emotion·category·labels·scene 설명** = 사건의 *맥락* |
| 돈(광고비) | `YT_bumper_cost_per_day`, `YT_bumper_cost_per_week` | channel_id+date | 일/주별 채널 광고비 |
| 세분 광고비 | `YT_brand_bumper_cost_per_day` | channel_id+date | 가로/세로(h/v) × 브랜드/스킵/논스킵/광고(b/s/ns/a)별 views·cost |
| 추세 | `YT_bumper_video_trend`, `YT_brand_video_trend` | video_id(+channel_id)+date | 날짜별 views·likes·comments |
| 산업 벤치마크 | `YT_bumper_cpm_by_industry` | industry | 산업별 bumper/skippable/non_skippable/average CPM, VTR |
| 캠페인 상태 | `YT_bumper_on_off` | video_id, channel_id | is_on·start·end |
| 크롤 운영 | `YT_bumper_crawl_video_list`, `YT_bumper_crawl_waiting_list` | video_id | added_date(신규 진입 탐지 보조) |

### ⚠️ 설계 전제 2가지 (현재 샘플의 한계)
1. **"오늘 vs 직전" 델타가 헤드라인의 본질.** 현재 50행 샘플은 정렬을 무시하고 앞부분만 받아서 한 테이블 안 날짜가 한 날로 쏠려 있음(예: `video_trend` 전부 `2024-08-01`, `cost_per_day` 전부 `2023-06-06`). → **실제 구현 시 `date DESC` 정렬로 최신일 + 비교 기준일(직전일 또는 직전 7일 평균)을 반드시 함께 받아야 함.**
2. 각 테이블의 "최신일"이 서로 다를 수 있음 → **기준 날짜 D = (트렌드/광고비 테이블의 최댓값 date)** 로 정의하고, 비교 기준 = D-1 또는 D의 직전 7일 평균.

## 3. 파이프라인 (4단계)

```
[1 적재/필터]      [2 신호 추출]            [3 사건 랭킹]        [4 LLM 작문]
date DESC 정렬  →  4종 신호별 델타·이상치  →  점수화·다양성 확보  →  어그로 헤드라인 5개
최신일 D + 기준  →  사건 후보 생성          →  Top 후보 셀렉      →  (수치 근거 동봉)
```

### 단계 1 — 적재 & 필터
- 모든 시계열 테이블을 `date DESC` 정렬해 **최신일 D**와 **비교 기준(D-1 / 7일 평균)** 구간만 적재.
- `channel_id → 브랜드명` 매핑 사전 구축 (`YT_bumper_ad.brand1` / `_global.brand` 우선).
- `video_id → {제목, 브랜드, 장르, emotion, category, ad_story}` 맥락 사전 구축(analized·scene 조인).

### 단계 2 — 신호 추출 (4종 전부)

**A. 광고비 급변** (`cost_per_day` + `brand_bumper_cost_per_day`)
- 채널/브랜드별 `cost(D) vs 기준` → 절대 증감액 & 증감률.
- 세분 테이블로 "무엇이 끌어올렸나"(세로형 스킵? 가로형 논스킵?) 분해.
- 출력 사건: `{brand, Δcost, Δ%, 주도 세그먼트}`

**B. 떡상 영상** (`video_trend` / `brand_video_trend`)
- video_id별 `views/likes/comments(D) vs 기준` 일간 증가량·증가율 → 바이럴 이상치 탐지.
- `video_id` → 제목·브랜드·**emotion·category·ad_story** 조인해 "어떤 광고가" 떡상했는지 맥락 부여.
- 출력 사건: `{video, brand, Δviews, Δ%, 맥락(감정/카테고리/스토리)}`

**C. 산업 CPM 변동** (`cpm_by_industry`)
- 산업별 bumper/avg CPM·VTR의 극단치(최고·최저), (시계열 확보 시) 전일 대비 변동.
- 출력 사건: `{industry, CPM, VTR, 순위/이상치}`

**D. 캠페인 on/off & 신규 진입** (`on_off` + `crawl_waiting_list`/`crawl_video_list`)
- `start == D` (오늘 켜짐) / `end == D` (오늘 꺼짐) 캠페인 탐지.
- `added_date == D` 신규 크롤 진입 영상 = 신규 광고주/소재 등장.
- 출력 사건: `{brand/video, on|off|신규, 시점}`

→ 모든 사건은 **공통 스키마**로 정규화:
```json
{ "type": "cost|momentum|cpm|campaign",
  "subject": "브랜드/영상/산업명",
  "metric": "지표명",
  "today": 값, "baseline": 값, "delta": 값, "delta_pct": 값,
  "context": "감정/카테고리/스토리 등 어그로용 맥락",
  "evidence": "출처 테이블·행 식별자" }
```

### 단계 3 — 사건 랭킹 & 다양성
- 신호별로 정규화 점수(=놀라움/크기: |Δ%| × 절대규모 로그) 계산.
- **다양성 강제**: 5개 헤드라인이 전부 "광고비 급증"이 되지 않도록, 타입별 상위에서 고르게 추려 Top ~15 후보 셀렉.
- 데이터 부족(예: 비교 기준 없음) 신호는 자동 제외(거짓 사건 방지).

### 단계 4 — LLM 작문
- **입력**: 기준 날짜 D + Top 사건 후보 JSON(수치+맥락). raw 행 금지.
- **시스템 프롬프트 핵심**:
  - 독자 = 마케터. 한국어 1줄, 5개, **서로 다른 사건**.
  - 어그로/후킹 강하게. 단, **제공된 수치 밖의 사실 창작 금지**(과장 표현 OK, 숫자 왜곡·허위 금지).
  - 길이 가이드(예: 공백 포함 ~40자 내), 클릭 욕구 자극 패턴(숫자·격차·반전).
  - 각 헤드라인은 어떤 사건(evidence)에 근거하는지 내부적으로 매핑.
- **출력(최종 산출물)**: 1줄 헤드라인 5개 텍스트. (근거 매핑은 내부 검증용으로만 보관, 사용자 노출은 헤드라인 5개만.)

## 4. 헤드라인 톤 예시 (감 잡기용, 실제 수치 아님)
- "어제 0원이던 △△, 하루 만에 광고비 3.2억 태웠다"

- "오늘 조용히 캠페인 끈 대형 브랜드 5곳, 누구일까"
- "신규 진입 광고주 12곳… '여기'가 시장에 칼 빼들었다"

## 5. 열린 이슈 (다음 단계에서 결정)
- 비교 기준: **D-1 단일** vs **직전 7일 평균** (노이즈 적은 후자 추천).
- LLM 모델/호출 방식 (Claude API 권장 — 캐싱 적용).
- 헤드라인 글자 수·금지어·브랜드 실명 노출 정책.
- `cpm_by_industry`에 date가 없음 → 시계열 변동을 보려면 외부 적재 필요 여부 확인.
- 자동 검증 단계: 생성된 헤드라인이 근거 사건의 수치를 위반하지 않는지 LLM/규칙으로 재검.
